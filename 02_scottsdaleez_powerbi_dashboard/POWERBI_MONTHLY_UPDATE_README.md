- Reference period: April 2026

## Purpose
This guide explains how to refresh the ScottsdaleEZ Power BI dashboard each month using the latest CSV outputs from the approved prediction pipeline.

This process assumes:
- the Power BI file (`.pbix`) has already been built
- the existing CSV filenames stay the same from month to month
- the dashboard visuals and data model do not need structural changes

---

## Monthly update goal
Each month, the City or project team should be able to:
1. run the prediction pipeline
2. replace the old CSV output files with the new ones
3. refresh the Power BI file
4. validate that the latest observed month and next prediction month display correctly
5. export or present the refreshed dashboard

## Recommended file management setup
Keep one folder that contains:
- the Power BI file (`ScottsdaleEZ_PowerBI_Dashboard_v1.pbix` or current working file)
- the latest CSV outputs

Recommended folder structure:

```text
ScottsdaleEZ_Dashboard/
├── ScottsdaleEZ_PowerBI_Dashboard_v1.pbix
├── predicted_monthly_volume_by_zip_requesttype.csv
├── monthly_volume_by_workgroup_zip.csv
```

Important:
- keep the filenames exactly the same
- overwrite the old CSVs with the new monthly run outputs
- do not rename columns unless the Power BI model is also updated

## Monthly update workflow

### Step 1. Run the prediction pipeline
Run the approved monthly prediction pipeline and confirm that the new CSV outputs were created.

Before moving to Power BI, verify that the pipeline completed successfully and that the expected output files exist.

---

### Step 2. Replace the old CSV files
Copy the new pipeline output CSVs into the dashboard folder.

When prompted:
- choose **Replace** or **Overwrite**
- keep the filenames exactly the same as the previous month

Do not:
- rename the files
- add month names into the filenames
- delete fields or reorder columns manually

The Power BI file is easiest to maintain when the filenames and schema remain stable.

---

### Step 3. Open the Power BI file
Open the existing dashboard `.pbix` file.

Example:
- `ScottsdaleEZ_PowerBI_Dashboard_v1.pbix`

Do **not** build a new Power BI file each month.
Use the existing file and refresh it.

---

### Step 4. Refresh the data
In Power BI Desktop:
1. open the `.pbix`
2. go to **Home**
3. click **Refresh**

Wait for all tables to refresh.

If Power BI asks you to confirm file locations, point it to the same folder where the updated CSVs are stored.

---

### Step 5. Confirm the data loaded correctly
After refresh, validate the following:

#### Predictions
- `prediction_month` reflects the next month after the most recent complete observed month
- the **Hotspot / Predictions** tab updates correctly
- the map, cards, table, and Issue Status slicer all respond consistently on the prediction tabs

#### Observed data
- `created_month` reflects the latest observed complete month
- the **Overview**, **Workgroup On-Target Performance**, **Request Completion Time**, and **Current vs Prior Year** tabs update correctly

#### Slicers
- ZIP slicers still work on ZIP-based tabs
- Month slicers still work on date-based tabs
- Workgroup slicers still work on workgroup tabs
- synced slicers still propagate across the intended tabs

## What to check on each tab after refresh

### 1. Overview
Check that:
- the cards populate
- the monthly trend line extends through the latest observed month
- the top request types chart updates correctly
- the staff-versus-citizen chart updates correctly
- ZIP and Month slicers still function

Files used:
- `seasonal_comparison_zip_requesttype.csv`
- `staff_vs_citizen_monthly_volume.csv`

Note:
The **Percent Change from Prior Year** metric works best when one month is selected. For example, January 2026 is compared with January 2025. When **All** is selected, it combines same-month prior-year comparisons across multiple months.

---

### 2. Hotspot / Predictions
Check that:
- the Azure Map loads ZIP predictions correctly
- the prediction card, ZIP count card, map, and table respond to the same filters
- the Issue Status slicer filters the page correctly
- the table is sorted by predicted request volume descending
- the forecast shown is for the current `prediction_month`

Files used:
- `predicted_monthly_volume_by_zip_requesttype.csv`

Note:
The current simplified dashboard uses the monthly prediction table directly for the main prediction views.

---

### 3. Workgroup On-Target Performance
Check that:
- workload by workgroup updates correctly
- the closed-within-14-days mix updates correctly
- the percent-closed-within-14-days chart updates correctly
- Workgroup and Month slicers still work

Files used:
- `monthly_volume_by_workgroup_zip.csv`
- `sla_performance_by_workgroup.csv`

---

### 4. Request Completion Time
Check that:
- the histogram uses the correct bin order:
  - `0-1d`
  - `1-3d`
  - `3-7d`
  - `7-14d`
  - `14-30d`
  - `30d+`
- the `(Blank)` bin is not showing
- the workgroup delay chart updates correctly
- Workgroup and Month slicers still work

Files used:
- `resolution_time_bins_by_workgroup.csv`

Note:
The current report uses a small resolution-bin lookup table inside Power BI to keep the bins in the correct order.
If the bin names change in the CSV, that lookup table must be updated too.

---

### 5. Current vs Prior Year
Check that:
- the cards update correctly
- the current vs prior year trend chart updates correctly
- the request-type change chart updates correctly
- ZIP and Month slicers still work

Files used:
- `seasonal_comparison_zip_requesttype.csv`

Note:
This tab is most interpretable when a specific month is selected. When **All** is selected, the prior-year comparison cards and chart reflect matched same-month comparisons aggregated across multiple months.

---

### 6. Leadership Watchlist
Check that:
- the hotspot table updates correctly
- the problem-area card behaves correctly under the selected Issue Status filter
- the YoY increase chart updates correctly
- the ZIP and Problem Status slicers still work

Files used:
- `predicted_monthly_volume_by_zip_requesttype.csv`
- `sla_performance_by_workgroup.csv`
- `seasonal_comparison_zip_requesttype.csv`

Note:
If **Within Normal Range** is selected, the **Number of Potential Problem Areas** card may go blank. That is expected because the card is only intended to count potentially rising forecast items.

Check that:
- the worksheet summaries still match the final tab names

- the caution text remains readable and unchanged after refresh

- **Prediction tabs** use the next `prediction_month`

## When a refresh is successful
A refresh is successful when:
- all visuals load without errors
- no CSV source is missing
- cards show values instead of blanks
- slicers work normally
- the latest observed month and next prediction month make sense
- the dashboard can be presented without manual rework

## If something breaks

### Problem: Power BI cannot find a file
Fix:
- point Power BI to the folder containing the updated CSVs
- confirm the filename still matches exactly

### Problem: a visual goes blank after refresh
Fix:
- check whether the visual is filtered to a month that no longer exists
- check whether a slicer selection is still active
- check whether a visual-level filter is conflicting with the slicer
- clear filters and test again

### Problem: the histogram bins are out of order
Fix:
- check the `DimResolutionBin` lookup table in Power BI
- confirm the relationship to `resolution_time_bins_by_workgroup[resolution_bin]` still exists
- confirm the sort order table still contains:
  - `0-1d`
  - `1-3d`
  - `3-7d`
  - `7-14d`
  - `14-30d`
  - `30d+`

### Problem: a slicer no longer syncs across tabs
Fix:
- go to **View → Sync slicers**
- confirm the correct pages are checked for **Sync** and **Visible**

### Problem: prediction visuals do not agree with each other
Fix:
- confirm the card, slicer, map, and table are all using the same prediction table
- confirm the Issue Status slicer is built from the same prediction table used by the page visuals
- confirm visual interactions are set to **Filter**, not **None**

### Problem: columns or fields are missing
Fix:
- compare the new CSV against the prior version
- if the pipeline schema changed, the Power BI model may need to be updated before refresh will work properly

## Final note
The easiest monthly update process is:
- keep the same file names
- keep the same column names
- overwrite the CSVs
- refresh the existing PBIX
- validate the tabs

If the pipeline logic or schema changes materially, the dashboard should be reviewed before the next monthly refresh cycle.
